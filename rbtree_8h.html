typedef int(* rbtree_cmp_keys_callback)(void *k1, size_t k1size, void *k2, size_t k2size)

Callback that, if provided, will be used to compare node keys. If not defined memcmp() will be used in the following way :

Parameters
k1The first key to compare
k1sizeThe size of the first key to compare
k2The second key to compare
k2sizeThe size of the second key to compare
Returns
The distance between the two keys. If the size is different 0 will be returned if the keys match (both size and value); "k1size - k2size" will be returned if the two sizes don't match; The difference between the two keys is returned if the two sizes match but the value doesn't
Note
By default memcmp() is be used to compare the value, a custom comparator can be registered at creation time (as parameter of rbtree_create())
If integers bigger than 8 bits are going to be used as keys, an integer comparator should be used instead of the default one (either a custom comparator or one of the rbtree_cmp_keys_int16(), rbtree_cmp_keys_int32() and rbtree_cmp_keys_int64() helpers provided by the library).
typedef int(* rbtree_walk_callback)(rbtree_t *rbt, void *key, size_t ksize, void *value, size_t vsize, void *priv)

Callback called for each node when walking the tree.

Parameters

vsizeThe size of the value
Returns
0 if the walker can go ahead visiting next node, any NON-ZERO integer if the walker should stop and return
